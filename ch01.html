数学の道具箱① ベクトル

「向きと長さ」を表す矢印、ベクトル。3D空間で光の計算をするなら、ベクトルは必須の道具です。

ベクトルとは「矢印」

ベクトルとは、向きと長さを持つ矢印のこと。 日常で例えるなら：

  • 「北に 10 km」という移動 ← 向き（北）と距離（10 km）
  • 「時速 80 km で東へ移動」という速度 ← 向き（東）と速さ（80 km/h）

コンピュータでは、ベクトルを3つの数 (x, y, z) で表すことが多いです。 例えば、「東に 3 歩、北に 4 歩、上に 5 歩」なら、v = (3, 4, 5) と書きます。

W1-a: ベクトル操作ツール

下のキャンバスで、矢印をドラッグして動かしてみてください。 右側に、ベクトルの成分と「長さ」がリアルタイム表示されます。

左：2D（xy平面）| 右：3D（xyz空間）

ベクトルの4つの基本操作

①足し算と引き算

2つのベクトル a = (1, 2, 3) と b = (2, 1, 1) があるとき：

a + b = (1+2, 2+1, 3+1) = (3, 3, 4)
意味：a だけ進んでから b だけ進んだのと同じ（移動の合成）
a − b = (1−2, 2−1, 3−1) = (−1, 1, 2)
意味：b の先端から a の先端へ向かうベクトル

②スカラー倍（伸び縮み）

ベクトル v = (1, 2, 3) に「2倍」をかけると：

2 · v = (2, 4, 6)
意味：同じ方向だが、長さが2倍になった矢印

③長さ（ノルム）の計算

ベクトル v = (3, 4) の長さは？ 3D座標の距離を計算するのと同じ（三平方の定理）：

|v| = √(3² + 4²) = √(9 + 16) = √25 = 5
意味：原点から (3, 4) への距離は 5

3D の場合も同じ：

|v| = √(x² + y² + z²)

④正規化（長さを1に統一）

ときどき「方向だけ欲しい」ことがあります。そのときは、長さで割ります：

normalized(v) = v / |v|
意味：元のベクトルを、その長さで割ると、方向は同じだが長さが1になる矢印

例えば v = (3, 4) で |v| = 5 なら：

normalized(v) = (3/5, 4/5) = (0.6, 0.8)
長さは √(0.6² + 0.8²) = √(0.36 + 0.64) = 1 になった

CGで使う4つの主要ベクトル

光の計算では、以下の4つのベクトルが出てきます。それぞれ正規化された方向です：

● n — 法線ベクトル（Normal）

表面の「向き」を表すベクトル。球の表面なら、その点で「一番外を向いている方向」です。 どの点の法線ベクトルを見ても、長さは1。

● l — ライト方向（Light direction）

光がどこからやってくるかを示すベクトル。 ライトが点 (2, 3, 4) にあり、サーフェス上の点が原点なら、 l は「原点から見たライトの方向」。これも正規化されています。

● v — 視線方向（View direction）

カメラ/目がどこにあるかを示すベクトル。「サーフェス上の点から、カメラへ向かう方向」です。 やはり正規化されています。

● h — ハーフベクトル（Half vector）

「l と v の真ん中の方向」を表します。 計算方法は h = normalize(l + v)。 光沢のある反射を計算するときに、「ハイライトが出ている角度」を判定するのに使われます。

W1-b: 球体上の四方向

右のミニビューポートで、球体上の任意の点をクリックしてみてください。 その点での n（法線）、l（ライト）、v（視線）、h（ハーフ） が矢印で表示されます。

スライダーでライト方向を動かすと、l が変わり、その結果 h も更新されます。

3つの大事な関係式

これから使う関係式の先取りです。まだ完全に理解する必要はありませんが、「こういう式がある」と覚えておいてください。

h = normalize(l + v)
ハーフベクトルは、ライト方向と視線方向を足して正規化したもの
l · n = 光がどれだけ当たるか（内積の話は次章）
ライト方向と法線が揃っていると大きな値、垂直だと0、逆向きだと負
v · h = ハイライトがどれだけ見えるか
視線とハーフが揃っていると、反射光が目に入る（次の第2章で詳しく）

まとめ

  • ベクトルは「向きと長さ」を表す矢印
  • 足し算・引き算・スカラー倍・正規化の4つの操作が基本
  • CGでは n（法線）、l（ライト）、v（視線）、h（ハーフ）という4つの方向ベクトルが常に登場する
