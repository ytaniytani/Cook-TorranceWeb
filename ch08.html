スペキュラ成分②: D項（法線分布関数 / GGX）

「h を向いた鏡はどれくらいあるか？」を答える関数。roughness がハイライトの形を決める仕組みが、ここで明らかになります。

式

DGGX(h) = α² / ( π · ( (n·h)²·(α²−1) + 1 )² )
α = roughness²
意味：マイクロファセットの向きが h と揃っている「密度」。n·h が1（完全に揃う）に近いほど大きい

入力は2つだけ。n·h（面の法線とハーフベクトルの揃い具合、第2章の内積）と、 α（roughness の2乗）です。

なぜ α = roughness² ？ スライダーの roughness をそのまま使うと、変化が 0 付近に偏って操作しづらいためです。 2乗してから使うと、スライダー全域で見た目が均等に変化します（Disney/UE4 以来の業界慣習）。

オリジナルの Cook-Torrance は GGX ではない（Beckmann 分布）

1982年の Cook & Torrance の原論文では、D に Beckmann 分布が使われていました：

DBeckmann(h) = exp( ((n·h)² − 1) / (m²·(n·h)²) ) / ( π · m² · (n·h)⁴ )
m は表面の粗さ。exp は「入力が 0 から離れるほど急激に小さくなる関数」という直感でOK（中学数学の範囲外なので詳細は割愛）

その後、2007年に Walter らが GGX（＝Trowbridge-Reitz, 1975年の再発見）を提唱し、 現在のゲームCG（UE・Unity・Substance）では GGX が事実上の標準になりました。

なぜ GGX が勝ったのか？ 答えは「裾」です。 Beckmann はハイライトの周囲が急激に暗くなりますが、GGX はぼんやりした光の尾を長く残します。 実測データや実物の見た目に近いのは GGX でした。

まとめ

  • D は「h を向いたマイクロファセットの密度」。ハイライトの形と鋭さを決める
  • α = roughness²。スライダーの効きを均等にするための業界慣習
  • GGX の個性は長い裾＝ハイライトの外のふわっとした光の尾
  • 原論文は Beckmann だったが、見た目のリアルさで GGX が標準になった
